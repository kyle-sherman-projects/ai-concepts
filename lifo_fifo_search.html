LIFO vs FIFO — DFS & BFS

Why the data structure defines the search strategy

The core insight: The only difference between DFS and BFS is which node gets popped from the frontier next. A LIFO stack always pops the most recently added node → you go deep first → that is DFS. A FIFO queue always pops the oldest node → you exhaust each level before going deeper → that is BFS. You cannot separate the strategy from the structure — they are the same thing.

WHY YOU CAN'T SWAP THEM

DFS + LIFO ✓ Always goes deepest first. The stack keeps the deepest unexplored path on top.
BFS + FIFO ✓ Always exhausts current depth before going deeper. The queue preserves level order.
DFS + FIFO ✗ A FIFO frontier forces level-by-level exploration — you get BFS behavior, not DFS.
BFS + LIFO ✗ A LIFO frontier always dives deep — you get DFS behavior, not BFS.
Bottom line: The data structure is the strategy. Swapping one changes the other automatically.
